Pure Chess Library Documentation

Engine Class

The Engine class in the Pure Chess Library provides the core logic for handling chess moves, validations, and game states.

Methods

MakeMove(move: Move): boolean

Attempts to make a move on the chess board. Returns true if the move is valid and successful, otherwise false.

This method internally calls the ValidateMove method to check if the move is legal before executing it. It also updates the game state and player turn accordingly.

ValidateMove(move: Move): boolean

Validates if a given move is legal according to the chess rules. Returns true if the move is valid, otherwise false.

This method checks the legality of the move based on the type of piece and the specific movement rules. It considers factors such as the color of the piece, the positions of other pieces, and special move conditions like castling and en passant.

ForceMove(initialSquare: Square, targetSquare: Square): void

Forces a move on the chess board, bypassing move validation. This method is primarily used for testing purposes.

Using this method allows you to directly modify the board state for debugging and testing purposes without adhering to the rules of chess. It can be useful for simulating specific positions or scenarios.

isInCheck(): boolean

Checks if the current player's king is in a check position.

This method iterates through all possible moves on the board and checks if any of them could capture the king of the current player. If such a move is found, it indicates that the king is in check, and the method returns true; otherwise, it returns false.

This method is crucial for detecting checkmate and stalemate conditions and for ensuring the validity of the game state.

UndoLastMove(): void

Undoes the last move made in the game.

This method is used to revert the game state to the previous position, allowing for move exploration and analysis. It's essential for implementing game analysis, AI, and debugging.

Properties

depth: number

The search depth used in the engine's AI move calculation. Can be adjusted to control the engine's playing strength.

Increasing the search depth allows the engine to explore more possible moves ahead, resulting in stronger play but also requiring more computational resources and time. The depth value can impact the engine's ability to find optimal moves in complex positions.
